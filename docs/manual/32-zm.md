# 32 Zusammenfassende Meldung (ZM)

> Pfad: Steuer → **Zusammenfassende Meldung**.
> Pflicht-Modul: Finanzamt-Export muss aktiv sein.

## Pflicht

Wenn Du **EU-Reverse-Charge-Rechnungen** ausstellst (Dienstleistungen an EU-Unternehmer-Kunden mit gültiger UID, USt-Schuld geht über § 3a Abs. 6 UStG / Art. 196 MwStSyst-RL auf den Empfänger über), bist Du verpflichtet, eine **Zusammenfassende Meldung** an FinanzOnline abzugeben.

**Fristen:**
- Monatsmeldung: bis Letzter des Folgemonats (Mai-Meldung bis 30.06.)
- Quartalsmeldung: bis Letzter des dem Quartal folgenden Monats (Q2 bis 31.07.)

## Was zeigt /at-tax/zm

Aggregation aller **finalisierten** Rechnungen mit `reverseCharge=true` und gesetzter `recipientVatId`, gruppiert pro Empfänger-UID. Land wird aus Customer-Billing-Address geholt, Fallback auf erste 2 Zeichen der UID.

**Filterung:**
- Period-Picker: Monat / Quartal / Jahr
- AT-Inland-RC wird ausgeklammert (selbe Steuerschuld, keine ZM-Pflicht)
- Drittland-RC ebenfalls (keine ZM-Pflicht, aber als Info angezeigt)

**Warnungen:**
- RC-Rechnungen ohne UID werden separat ausgewiesen — UID am Customer oder an der Rechnung nachtragen vor Meldung.

## CSV-Download

Format kompatibel mit FinanzOnline:
```
UID;Betrag (EUR);Dreiecksgeschaeft;Rechnungs-Nr.
ATU12345678;1.234,56;nein;0042,0045
DE123456789;567,89;nein;0050
```

CSV in FinanzOnline hochladen → Felder werden vorgefüllt → User prüft + bestätigt.

## Dreiecksgeschäfte

Sind in der Auto-Generierung **nicht** abgebildet. Wenn Du als Mittler in einem Dreiecksgeschäft beteiligt bist (Lieferung von A nach C über Dich in B), musst Du die Spalte „Dreiecksgeschäft" beim Upload manuell auf „ja" setzen.
